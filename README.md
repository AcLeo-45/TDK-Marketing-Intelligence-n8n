# TDK Marketing Intelligence Automation Workflow

![TDK](https://img.shields.io/badge/TDK-Case%20Study-blue) ![n8n](https://img.shields.io/badge/Built%20with-n8n-orange) ![MySQL](https://img.shields.io/badge/Database-MySQL-blue) ![Gmail](https://img.shields.io/badge/Report-Gmail-red)

## 📌 Overview

A **Marketing Intelligence Automation Workflow** built using **n8n** that automatically ingests marketing data from a MySQL database, calculates key business metrics, and delivers a professionally formatted HTML report via email — every day at 8:00 AM automatically.

---

## 🎯 Objective

Build an end-to-end automated workflow that:
- Pulls campaign, leads, spend, and conversion data from MySQL
- Calculates CAC, ROI, Conversion Rate, and Lead Quality Score
- Sends a single consolidated daily email report with all metrics

---

## 🛠️ Tech Stack

| Tool | Purpose |
|------|---------|
| **n8n** (Cloud) | Workflow orchestration and automation |
| **MySQL** (Railway) | Cloud-hosted relational database |
| **Gmail** (OAuth2) | Automated email report delivery |
| **JavaScript** | Metric calculations inside n8n Code node |

---

## 📊 Database Schema

| Table | Description |
|-------|-------------|
| `users` | Company accounts, regions, departments |
| `campaigns` | Campaign name, channel, budget, status |
| `leads` | Lead info, quality score, source, campaign |
| `conversion_events` | Clicks, purchases, demo requests, event value |
| `marketing_spend` | Daily spend, impressions, clicks, conversions |

> Full column-level data definitions available in `Data_Definitions.csv`

---

## ⚙️ Workflow Structure

```
Schedule Trigger (Daily 8:00 AM)
        ↓
MySQL Node 1 — Fetch Campaigns + Spend Data (8 items)
        ↓
MySQL Node 2 — Fetch Leads + Revenue Data (64 items)
        ↓
Code Node (JavaScript) — Calculate CAC, ROI, Conversion Rate
        ↓
Gmail Node (Execute Once) — Send Single HTML Email Report (1 item)
```

---

## 📐 Metrics Calculated

| Metric | Formula | Purpose |
|--------|---------|---------|
| **CAC** | Total Spend ÷ Total Conversions | Cost to acquire one customer |
| **ROI** | (Revenue - Spend) ÷ Spend × 100 | Return on marketing investment |
| **Conversion Rate** | Conversions ÷ Clicks × 100 | Campaign effectiveness |
| **Avg Lead Quality Score** | AVG(lead_quality_score) per campaign | Lead quality per channel |

## 🚀 How to Run

### 1. Setup Database
- Create a free MySQL database on [Railway.app](https://railway.app)
- Go to **Database → Query tab**
- Run `raw.sql` to create all tables and insert sample data

### 2. Import Workflow into n8n
- Open your n8n instance
- Go to **Workflows → Import**
- Upload `TDK_Marketing_Intelligence_Workflow.json`

### 3. Configure Credentials
- Add your **MySQL credentials** (host, port, user, password, database name)
- Connect **Gmail account** via OAuth2
- Update the **"To" email** in Gmail node to your stakeholder email

### 4. Activate Workflow
- Click **Publish** in n8n cloud
- Workflow runs automatically every day at **8:00 AM**
- One consolidated email report will be sent per run

---

## 💡 Assumptions

- Sample data from `raw.sql` is used for demonstration purposes
- In a production environment, live data would be inserted into MySQL daily — resulting in updated metrics each run
- ROI shows -100% for campaigns with no purchase conversion events in sample data — expected behavior with provided dataset
- Gmail node uses **Execute Once** setting to send a single email per workflow run regardless of item count
- Email recipient is configurable — update the Gmail node "To" field as needed
